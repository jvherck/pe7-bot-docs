PE7 Discord Bot

PE7 Bot is the bot for Prospective Esports' Discord server. The bot serves multiple purposes: moderation, logging, socials integrations, server management, support system, ... There's new features being added all the time.
These docs are meant to make you a little bit more comfortable with using this bot and discovering its capabilities. To see what has changed, you can switch between older and newer versions of the bot through the Versions dropdown menu.

Invoking commandsâ€‹

When invoking commands it's very important to follow the given order in the argumented command table! Mixing the arguments can lead to misformed output or errors. If you don't wish to use one or more of the arguments, simply don't include them in your command (if there are no other arguments after the one you want to skip) or use the default value which you can find in the argumented command table.
Arguments are seperated by spaces!

Default valuesâ€‹

Some commands have arguments with default values. If you don't include your wanted value for that argument in your command, the bot will automatically use this default value to continue. Default values can be found in the argumented command table. The default value None indicates no default value is present because the argument does not need a default value (e.g. twitch doesn't have a default value because it does not need to show a specific streamer if no argument is present in your command, it will just show which streamers are currently live).

Terminologyâ€‹

  • Empty command: when a command is sent without any arguments (e.g. twitch would be an "empty command" while twitch prospectiveesports would be an "argumented command").
  • Argumented command: when a command is sent with one or more arguments (see example above).
  • Alias: an alias is another name to invoke the same command (e.g. commands is an alias for help).
  • Argument: an argument is an extra parameter that gives the bot more info on what to do (e.g. help [command or category] would not just show the general help menu, but show the help menu specifically for the given command or category).
  • Type: tells you what type of info the argument requires (e.g. mentioning a channel for an argument with type "discord username/id" will not work).
